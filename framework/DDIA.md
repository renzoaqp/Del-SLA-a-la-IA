# DDIA Framework

DDIA es un metodo para disenar servicios tecnologicos basados en datos.

## 1. Define
Definir el servicio antes de medirlo.

Preguntas clave:
- Que servicio se entrega?
- Quien es el usuario?
- Cual es el valor de negocio?
- Cual es el horario de servicio?
- Que esta incluido y que esta excluido?

## 2. Design
Disenar indicadores y objetivos.

Componentes:
- SLI: como se mide.
- SLO: objetivo interno.
- SLA: compromiso externo.
- KPI: indicador de gestion.
- OKR: objetivo estrategico con resultados clave.

## 3. Instrument
Instrumentar la medicion.

Elementos:
- Fuente de datos.
- Frecuencia de medicion.
- Ventana de calculo.
- Dashboard.
- Validacion.
- Propietario del dato.

## 4. Analyze
Analizar para decidir.

Resultados esperados:
- Brechas entre objetivo y realidad.
- Causas probables.
- Decisiones operativas.
- Mejoras de proceso.
- Automatizacion.
- Casos de uso de IA.

## Principio central
Sin datos confiables no hay SLA confiable. Sin SLA confiable no hay gobierno del servicio.
