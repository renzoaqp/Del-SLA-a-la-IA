## Operating principles
- Do not invent references.
- Prefer scientific, technical and primary sources.
- Blogs can inspire but cannot be used as evidence.
- Separate evidence, interpretation and opinion.
- Keep all claims traceable to sources or implementation data.
- Write in professional Spanish unless a file requires English.
- Maintain consistency with DDIA.

## Project goal
Build a practical and research-ready framework for IT service engineering based on data.

## DDIA
- Define: define service, users, scope, value and constraints.
- Design: design SLI, SLO, SLA, KPI and OKR.
- Instrument: define data sources, metrics, dashboards and validation.
- Analyze: interpret results, decisions, improvements and AI use cases.

## Output quality
Every article, framework section or research note must answer:
- What problem does it solve?
- What data is required?
- How is it measured?
- What decision does it enable?
- What evidence supports it?
